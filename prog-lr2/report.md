# Лабораторная работа 2. Методы оптимизации вычисления кода с помощью потоков, процессов, Cython, отпускания GIL

## Цель работы: исследовать методы оптимизации вычисления кода, используя потоки, процессы, Cython и отключение GIL на основе сравнения времени вычисления функции поиска простых чисел, реализованной на чистом Python.

## Структура проекта
prog-lr2/
├── main.py               # Главный скрипт с 5 итерациями оптимизации
├── prime_functions.pyd   # Скомпилированный Cython-модуль (бинарное расширение)
├── prime_functions.pyx   # Исходный код на Cython (оптимизированные функции)
├── setup.py              # Скрипт компиляции Cython (.pyx → .pyd)
└── test.py               # Юнит-тесты для проверки функций

## Вывод

Многопоточность не дала значительного ускорения из-за GIL, который не позволяет потокам выполняться параллельно на нескольких ядрах для CPU‑bound задач.

Многопроцессность обходит GIL путём создания отдельных процессов, каждый со своим интерпретатором, и даёт ускорение.

Cython компилирует Python‑подобный код в C, что даёт уже 15.16× ускорения даже на одном ядре.

NOGIL использует параллельные вычисления без GIL, достигая максимального ускорения 21.22×.
