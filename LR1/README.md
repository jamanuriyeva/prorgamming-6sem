# LR1 Деплой статического сайта на Hugo через SourceCraft

**Цель:** Реализация CI/CD-процесса для автоматической сборки и публикации сайта на Hugo с дополнительными проверками качества кода.

---

## 1. Регистрация и настройка инфраструктуры

- **Регистрация:** Учетная запись Yandex успешно использована для входа в SourceCraft.

---

## 2. Настройка доступа (Токены и ключи)

Для безопасной работы с репозиторием и CI/CD были созданы следующие сущности:

- **Personal Access Token (PAT):**  
  Сгенерирован по [инструкции](https://sourcecraft.dev/portal/docs/en/sourcecraft/security/pat). Используется для аутентификации в системе и запуска пайплайнов.

- **SSH-ключи:**  
  Пара ключей (публичный и приватный) создана согласно [руководству](https://sourcecraft.dev/portal/docs/en/sourcecraft/security/ssh). Публичный ключ добавлен в настройки профиля SourceCraft для возможности клонирования по SSH.

---

## 3. Наполнение репозитория

В репозиторий был запушен базовый шаблон статического сайта на Hugo.  
Структура включает:

- `config.toml` — конфигурация сайта.
- Папки `content/`, `layouts/`, `static/` — стандартные директории Hugo.
- Файлы `.gitignore` и `README.md`.

Репозиторий клонирован и синхронизирован с удалённым сервером SourceCraft.

---

## 4. Реализация CI/CD пайплайна

Основной пайплайн переписан для среды SourceCraft (используется синтаксис YAML для `.sourcecraft/pipeline.yaml`).  
В результате реализован следующий процесс:

1. **Триггер:** Пуш в ветку `main` (или создание Pull Request).
2. **Сборка:** Установка Hugo, сборка статики в папку `public/`.
3. **Деплой:** Публикация содержимого `public/` в среду хостинга (согласно [туториалу](https://sourcecraft.dev/portal/docs/ru/sourcecraft/tutorials/sites)).
4. **Проверки:** Добавлены дополнительные шаги для контроля качества контента.

---

## 5. Дополнительные проверки Markdown

В пайплайн интегрированы отдельные шаги для проверки синтаксиса Markdown-файлов:

- **Шаг 1:** Установка линтера `markdownlint-cli`.
- **Шаг 2:** Запуск проверки всех `.md` файлов в папке `content/` с использованием правил по умолчанию.
- **Шаг 3:** (Опционально) Проверка орфографии с помощью `vale` или аналогичного инструмента.

Эти шаги выполняются до стадии сборки, что предотвращает публикацию сайта с синтаксическими ошибками в тексте.

---

## 6. Результат

- Сайт публикуется по адресу, но некорректно отображается
- Пайплайн автоматически запускается при каждом обновлении кода.
- Markdown-файлы валидируются на этапе CI, что гарантирует однородность контента.
